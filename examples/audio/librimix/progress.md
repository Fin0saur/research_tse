🚀 TSE 探针联合训练系统架构方案 (Architecture Design)1. 核心设计理念数据驱动 (Data-Centric)：将“硬负样本挖掘 (Hard Negative Mining)”与“正负例平衡 (Balancing)”作为提升模型判别能力的核心手段。物理意义打分 (Physical Scoring)：不依赖特征空间的聚类，而是通过 BSRNN 的真实前向推理，计算客观的 SI-SNRi 分数，以此作为“成功(C0)”与“失败(C1)”的唯一判据。静态采样与动态更新分离：在数据预备阶段静态打分以支持 InfiniteBalancedSampler，在联合训练阶段解冻 BSRNN 实现多任务动态优化。2. 系统模块划分整个系统分为三大独立模块，按顺序依次执行。每个模块的输入和输出必须有严格的接口约定。📦 模块 A：数据扩充器 (Data Expander)目标：将少量的原始混合音频（Mix）扩展为海量的“陷阱”组合，为模型提供充足的容错题库。输入 (Input)：raw.list (JSONL 格式)：包含原始的混合音频信息 (spk, wav_mix, target)。audio.json：全量声纹字典，记录每个说话人的所有干净语音路径。核心逻辑：纯文本处理。遍历 raw.list，为每个 Mix 根据目标说话人 ID 从字典中随机抽取 $N$ (通常为 20) 条不同的语音作为 Enrollment Cue。输出 (Output)：bias_enroll20.jsonl：扩充后的数据集（体积变大 $N$ 倍），增加了 audio_spk1 等 cue 路径字段，但不包含任何分数。⛏️ 模块 B：在线扫雷打分器 (Mining & Scoring Engine)目标：利用冻结的基线模型，对庞大的扩充数据集进行物理评测，甄别出哪些组合是真正的“硬骨头”。输入 (Input)：bias_enroll20.jsonl (模块 A 的输出)。基线 BSRNN 的预训练权重 (.pt 文件)。核心逻辑：加载并彻底冻结 BSRNN。遍历 JSONL，读取音频进行前向推理：out = model(mix, cue)。计算 Dynamic_SISNRi = cal_SISNR(out, target)。打标签：若 Dynamic_SISNRi < 1.0 则 label = 1 (C1 混淆/失败)，否则 label = 0 (C0 成功)。工程要求：必须支持多卡并行 (Sharding)。数据量极大，需通过切片并行推理，最后 cat 合并结果。输出 (Output)：train_scored.jsonl：在模块 A 的基础上，追加了 "Dynamic_SISNRi" 和 "label" 字段。🧠 模块 C：多任务联合训练器 (Joint Fine-tuning Trainer)目标：利用平衡的难易样本，同时优化目标提取模型（BSRNN）和质量诊断探针（Probe）。输入 (Input)：train_scored.jsonl (模块 B 的输出)。基线 BSRNN 权重。核心逻辑：采样策略：加载 train_scored.jsonl 提取全局的 label 列表。初始化 InfiniteBalancedSampler，根据 1/频率 设置权重，确保每个 Batch 抽取的 C0 和 C1 比例接近 1:1。网络状态：解冻 BSRNN（可训练），冻结声纹编码器（只读），初始化 CNN 探针（可训练）。前向挂载：在 BSRNN 的指定浅层（如 sep_layer_2 或 usef.post）挂载 Hook，截获 pmap 激活矩阵。双轨推理：音频过 BSRNN 得到分离波形 out_dynamic；截获的 pmap 过探针得到分类对数几率 logits。联合损失计算 (Multi-task Loss)：$$L_{total} = L_{sep\_loss}(out\_dynamic, target) + \lambda \cdot L_{probe\_cls}(logits, label_{static})$$注：$label_{static}$ 来自于文件里的初始打分，而非当前跑出来的分数，以稳定探针学习。$\lambda$ (如 0.1) 用于控制分类 Loss 不至于冲垮 BSRNN 的波形重建能力。输出 (Output)：联合优化后的双模型权重：joint_bsrnn_epX.pt 和 joint_probe_epX.pt。3. 数据流转规范 (Data Contract)这是整个流水线的“命脉”。不管具体的代码怎么写，这三个文件的数据结构约定决不能打破。D1: 基础数据输入格式 (raw.list)JSON{"key": "mix001", "spk": ["spkA"], "wav_mix": "/path/mix.wav", "wav_target":
